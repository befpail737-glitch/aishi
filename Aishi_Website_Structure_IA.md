# Aishi Distributor 英文网站 - 网站结构和信息架构

## 1. 信息架构概述

### 1.1 架构目标
- 为全球工程师和采购人员提供清晰的信息访问路径
- 有效展示Aishi产品线和解决方案
- 建立专业可靠的技术资源中心
- 优化用户转化和商务咨询

### 1.2 架构原则
- **用户导向**: 以用户需求和行为模式为基础设计
- **层级清晰**: 信息层级结构明确，易于理解
- **导航直观**: 提供多种导航方式，确保信息易发现
- **内容关联**: 相关内容之间建立有效连接

## 3. 导航系统设计

### 3.1 主导航 (Primary Navigation)
- **位置**: 页眉顶部
- **结构**: 水平排列，7个主要分类
- **移动端**: 折叠式汉堡菜单
- **固定导航**: 页面滚动时保持可见

### 3.2 辅助导航 (Secondary Navigation)
- **产品中心**: 垂直侧边栏，展示6大产品分类
- **面包屑导航**: 显示当前位置路径
- **页脚导航**: 包含重要页面快速链接

### 3.3 内容相关导航
- **相关产品**: 页面底部显示相关内容
- **分类浏览**: 产品页内快速切换产品类型
- **标签系统**: 技术文章按主题分类浏览

## 4. 用户路径设计

### 4.1 工程师用户路径
1. **技术需求路径**:
   首页 → 产品中心 → 特定产品类别 → 型号表格/技术参数
   
2. **解决方案路径**:
   首页 → 解决方案 → 相关行业方案 → 技术详情 → 联系我们

3. **文档查找路径**:
   首页 → 技术支持 → 选型指南/应用笔记 → 具体文档

### 4.2 采购人员路径
1. **产品采购路径**:
   首页 → 产品中心 → 产品分类 → 询价/库存查询

2. **供应商评估路径**:
   首页 → 关于我们 → 资质证书 → 联系我们

### 4.3 管理决策路径
1. **合作伙伴评估**:
   首页 → 关于我们 → 公司实力 → 联系我们

2. **行业趋势了解**:
   首页 → 新闻中心 → 行业动态 → 深度分析

## 5. 内容组织策略

### 5.1 内容分层
- **第一层**: 高层级概念和价值主张 (首页、关于我们)
- **第二层**: 产品/服务分类 (产品中心、解决方案)
- **第三层**: 详细规格和实现 (产品详情、方案详情)
- **第四层**: 深度技术内容 (技术文章、新闻详情)

### 5.2 内容标签系统
- **技术标签**: 低ESR、高纹波、长寿命、LED驱动、PD快充、变频器
- **产品标签**: 引线型、贴片型、固态电容、薄膜电容
- **应用标签**: 照明、电源、工业、新能源、汽车

### 5.3 内容关联策略
- **产品→技术文档**: 产品页链接到相关技术指南
- **技术→产品**: 技术文章中提及产品并链接
- **新闻→解决方案**: 行业动态关联到解决方案
- **解决方案→产品**: 方案推荐包含产品链接

## 6. 信息检索系统

### 6.1 全局搜索
- **搜索框位置**: 页眉右侧
- **搜索范围**: 全站内容，包括产品型号、技术参数、文章标题
- **搜索结果**: 按相关性排序，包含摘要和分类标识

### 6.2 筛选系统
- **产品筛选**: 电压、容量、温度、尺寸、纹波电流等参数筛选
- **内容筛选**: 按标签、日期、内容类型筛选
- **解决方案筛选**: 按应用领域、行业、技术特性筛选

## 7. 信息更新维护

### 7.1 内容生命周期
- **新闻时效性**: 最新新闻优先展示
- **产品数据更新**: 与Aishi官方数据同步更新
- **技术文档维护**: 定期审核技术准确性

### 7.2 版本控制
- **内容版本**: 重要技术文档标注更新日期
- **变更记录**: 重大内容更新记录变更日志
- **历史版本**: 关键页面保留重要历史版本

## 8. 可扩展性考虑

### 8.1 未来扩展路径
- **新产品线**: 预留添加新Aishi产品类别的结构
- **新应用领域**: 解决方案部分可扩展到新行业
- **多语言**: 架构支持未来多语言版本

### 8.2 模块化设计
- **组件复用**: 设计可复用的页面组件
- **内容模块**: 内容以模块形式组织，便于组合
- **功能模块**: 功能相对独立，便于扩展和维护
